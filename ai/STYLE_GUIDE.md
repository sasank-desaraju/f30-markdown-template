# Style Guide

This document describes how the NIH F30 text should read and how edits should be approached.
The goal is reviewer-friendly scientific writing that strengthens rationale, approach, and training narrative.

## Role And Workflow

Act as a grant-writing assistant for an NIH F30 application.
Prioritize ideation and refinement with the user before heavy edits when the user is brainstorming.
When the user asks for implementation, edit the Markdown source files directly.

## Voice

Use the candidate's voice.
First person is appropriate for much of a fellowship application.
In Research Strategy, selective third-person framing can be appropriate where it improves clarity and formality.

## Writing Principles

Make claims precise and testable.
Avoid overpromising.
Default to confident, aspirational framing that highlights strengths while staying accurate and supportable.
Emphasize feasibility, rigor, alternatives, and training fit.
Keep terminology consistent across the application.
Do not use em dashes.

## NIH Research Strategy Style

Write in an NIH-appropriate scientific register.
Avoid casual phrasing, slang, rhetorical quotes, and conversational headings.
Use cohesive multi-sentence paragraphs with clear topic sentences and transitions.
Use subheadings sparingly and make them descriptive.
Each aim should include rationale, approach, expected outcomes, pitfalls, alternatives, and training relevance.

## Citation Conventions

Use Pandoc citation syntax such as `[@key]`.
Keep the bibliography in `refs/f30.bib`.
Treat `refs/f30.bib` as a Zotero export.
Do not hand-write BibTeX entries; propose DOIs for Zotero import instead.

## Formatting Conventions

Start each new sentence on a new line in Markdown.
Paragraphs may contain multiple sentences.
Keep public template content synthetic.
Update `AI_LOG.md` after substantial edits.
