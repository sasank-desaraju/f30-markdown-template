# Writing Loop

1. Read `ai/PROJECT_INFO.md` and the target `markdown/` section.
2. Make the smallest useful Markdown edit.
3. Export the section to DOCX if a human needs to comment.
4. Reconcile feedback back into Markdown.
5. Update `AI_LOG.md` when a meaningful revision is complete.

Keep planning text separate from rendered sections when possible.
For example, use files named `*_planning.md` for rough notes and include only polished sections in `draft.qmd`.
